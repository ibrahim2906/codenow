C++ Focus | للمبرمج المبتدئ والمتوسط

هذا المرجع مُصمَّم ليكون Core Reference لأي مبرمج مبتدئ → متوسط، خصوصًا في C++ (وقابل للتطبيق على باقي اللغات).

أولًا: مصطلحات أساسية جدًا (Core Programming Concepts)

1️⃣ Program
Software / Application

التعريف: مجموعة أوامر مكتوبة بلغة برمجة لتنفيذ مهمة محددة.

#include <iostream>
using namespace std;

int main() {
    cout << "Hello World";
    return 0;
}

الأهمية:
  • هو الناتج النهائي الذي يتعامل معه المستخدم.
2️⃣ Algorithm
Logic / Procedure

التعريف: خطوات منطقية مرتبة لحل مشكلة.

int a, b;
cin >> a >> b;
cout << a + b;


الأهمية:
  • البرنامج الجيد أساسه خوارزمية صحيحة.
3️⃣ Variable
Identifier

التعريف: مكان في الذاكرة لتخزين قيمة قابلة للتغيير.

int age = 20;
age = 21;
الأهمية:
  • بدون متغيرات لا يمكن تخزين أو تعديل البيانات.
4️⃣ Constant
Read-only value

التعريف: قيمة ثابتة لا يمكن تغييرها بعد تعريفها.

const float PI = 3.14;
الأهمية:
  • يقلل الأخطاء.
  • يجعل الكود أوضح.
5️⃣ Data Type
Type

التعريف: يحدد نوع البيانات المخزنة في المتغير.

int x = 10;
float y = 3.5;
char c = 'A';
bool isOpen = true;
الأهمية:
  • يحدد حجم الذاكرة.
  • يحدد العمليات الممكنة.

ثانيًا: التحكم في سير البرنامج (Control Flow)

6️⃣ Condition
Decision

التعريف: تنفيذ كود بناءً على شرط.

if (age >= 18) {
    cout << "Adult";
}


الأهمية:
  • اتخاذ قرارات داخل البرنامج.
7️⃣ Loop
Iteration

التعريف: تكرار تنفيذ كود طالما الشرط متحقق.

for (int i = 0; i < 5; i++) {
    cout << i << endl;
}


الأهمية:
  • تنفيذ المهام المتكررة بكفاءة.
8️⃣ Break
Exit loop

التعريف: إنهاء الحلقة فورًا.

for(int i = 0; i < 10; i++){
    if(i == 5) break;
}
9️⃣ Continue
Skip iteration

التعريف: تخطي التكرار الحالي والانتقال للتالي.

for(int i = 0; i < 5; i++){
    if(i == 2) continue;
    cout << i;
}

ثالثًا: الدوال (Functions)

🔟 Function
Method (في OOP)

التعريف: كود قابل لإعادة الاستخدام.

int add(int a, int b) {
    return a + b;
}
1️⃣1️⃣ Parameter vs Argument

Parameter: متغير في تعريف الدالة.
Argument: القيمة المرسلة عند الاستدعاء. 

add(3, 4); // 3 و 4 = Arguments
1️⃣2️⃣ Return
Output

التعريف: القيمة التي تُعيدها الدالة.

return result;

رابعًا: البرمجة الكائنية (OOP)

1️⃣3️⃣ Class
Blueprint

التعريف: قالب لإنشاء كائنات.

class Car {
public:
    int speed;
};
1️⃣4️⃣ Object
Instance

التعريف: نسخة من Class.

Car myCar;
myCar.speed = 120;
1️⃣5️⃣ Encapsulation
Data Hiding

التعريف: إخفاء البيانات ومنع الوصول المباشر لها.

class Car {
private:
    int speed;
public:
    void setSpeed(int s) { speed = s; }
};
1️⃣6️⃣ Inheritance
Reusability

التعريف: كلاس يرث خصائص وسلوك كلاس آخر.

class SportsCar : public Car {};
1️⃣7️⃣ Polymorphism
Multiple Forms

التعريف: نفس الدالة بسلوك مختلف.

class Animal {
public:
    virtual void sound() {
        cout << "Animal sound";
    }
};

خامسًا: الذاكرة والمؤشرات

1️⃣8️⃣ Memory
RAM

التعريف: مكان تخزين البيانات أثناء التشغيل.

1️⃣9️⃣ Pointer
Address Holder

التعريف: متغير يخزن عنوان متغير آخر.

int x = 5;
int* p = &x;
2️⃣0️⃣ Reference
Alias

التعريف: اسم آخر لنفس المتغير.

int x = 10;
int& r = x;

سادسًا: فروق مهمة جدًا

🔴 Variable vs Constant
Variable Constant
تتغير لا تتغير
مرنة آمنة
🔴 Function vs Method
  • Function: خارج class
  • Method: داخل class

🔴 Parameter vs Argument
  • Parameter: في التعريف
  • Argument: عند الاستدعاء

🔴 Loop vs Condition
  • Loop: تكرار
  • Condition: قرار

🔴 Pointer vs Reference
Pointer Reference
يمكن تغييره لا
قد يكون NULL لا
🔴 Compile-time vs Run-time
  • Compile-time: قبل التشغيل
  • Run-time: أثناء التشغيل

سابعًا: مصطلحات متقدمة في التحكم والتعامل مع البيانات

2️⃣1️⃣ Scope (نطاق الرؤية)
Visibility Range

التعريف: يحدد أين يمكن استخدام المتغير داخل البرنامج.

الأنواع:

  • Local Scope: داخل دالة أو بلوك
  • Global Scope: خارج جميع الدوال
int x = 10; // Global

void test() {
    int y = 5; // Local
    cout << x; // ✔ مسموح
}
// cout << y; ❌ خطأ


لماذا مهم؟
  • يمنع التعارض بين الأسماء
  • يقلل الأخطاء
  • يجعل الكود أسهل قراءة
2️⃣2️⃣ Lifetime (عمر المتغير)
Variable Duration

التعريف: الفترة الزمنية التي يبقى فيها المتغير موجودًا في الذاكرة. 

void func() {
    int x = 5; // يُحذف بعد انتهاء الدالة
}
توضيح مهم:
  • Scope = أين أستخدم المتغير
  • Lifetime = متى يُحذف من الذاكرة
2️⃣3️⃣ Static
Persistent Variable

التعريف: متغير يحتفظ بقيمته حتى بعد انتهاء الدالة.

void counter() {
    static int count = 0;
    count++;
    cout << count << endl;
}

عند استدعاء الدالة عدة مرات → القيمة لا تُعاد للصفر.

الاستخدام الشائع:
  • العدّادات
  • حفظ الحالة
2️⃣4️⃣ Enum
Enumeration

التعريف: مجموعة قيم ثابتة ذات أسماء.

enum Status { Success, Error, Loading };
Status s = Success;
لماذا نستخدمه؟
  • بديل أوضح من الأرقام
  • يقلل الأخطاء
2️⃣5️⃣ Struct
Data Structure

التعريف: تجميع متغيرات مختلفة داخل كيان واحد.

struct Person {
    string name;
    int age;
};

Person p;
p.name = "Ali";
p.age = 20;
الفرق بين struct و class:
  • struct: public افتراضيًا
  • class: private افتراضيًا

ثامنًا: الذاكرة وإدارتها (Memory Management)

2️⃣6️⃣ Stack

التعريف: ذاكرة تلقائية وسريعة.

void func() {
    int x = 5; // Stack
}
مميزاته:
  • سريع جدًا
  • يُدار تلقائيًا
عيوبه:
  • حجم محدود
2️⃣7️⃣ Heap

التعريف: ذاكرة ديناميكية تُدار يدويًا.

int* p = new int(10);
delete p;

تحذير: نسيان delete يسبب Memory Leak.

2️⃣8️⃣ Memory Leak

التعريف: حجز ذاكرة دون تحريرها.

int* p = new int(5);
// لا يوجد delete
النتيجة:
  • بطء البرنامج
  • استهلاك الذاكرة
2️⃣9️⃣ Smart Pointer

التعريف: مؤشر ذكي يدير الذاكرة تلقائيًا.

#include <memory>
unique_ptr<int> p = make_unique<int>(5);
لماذا أفضل؟
  • لا Memory Leak
  • أكثر أمانًا

تاسعًا: OOP بشكل أعمق

3️⃣0️⃣ Abstraction

التعريف: إظهار ما يهم المستخدم فقط.

مثال تشبيهي: قيادة السيارة دون معرفة تفاصيل المحرك.

class Shape {
public:
    virtual double area() = 0;
};
3️⃣1️⃣ Constructor

التعريف: دالة تُستدعى تلقائيًا عند إنشاء الكائن.

class Car {
public:
    int speed;
    Car() {
        speed = 0;
    }
};
3️⃣2️⃣ Destructor

التعريف: دالة تُستدعى عند تدمير الكائن.

~Car() {
    cout << "Destroyed";
}
3️⃣3️⃣ Virtual Function

التعريف: دالة يتم تحديدها وقت التشغيل.

class Base {
public:
    virtual void show() {
        cout << "Base";
    }
};
3️⃣4️⃣ Override

التعريف: إعادة تعريف دالة موروثة.

class Derived : public Base {
public:
    void show() override {
        cout << "Derived";
    }
};
3️⃣5️⃣ Interface (في C++)

التعريف: كلاس يحتوي دوال virtual فقط.

class Flyable {
public:
    virtual void fly() = 0;
};
3️⃣6️⃣ Composition

التعريف: كلاس يحتوي كائن من كلاس آخر.

class Engine {};
class Car {
    Engine e;
};
أفضل من الوراثة عندما:
  • العلاقة ليست is-a

vector<int> v = {1,2,3};

auto sum = [](int a, int b) {

using namespace std;
cout << "Hello";

حادي عشر: مصطلحات أساسية مكمّلة (لا غنى عنها)

4️⃣1️⃣ Compile-time vs Run-time

Compile-time (وقت الترجمة):

  • المرحلة التي يترجم فيها الكود
  • يتم اكتشاف أخطاء الصياغة (Syntax)
int x = "Ali"; // نوع خاطئ (Compile-time)

Run-time (وقت التشغيل):

  • أثناء تنفيذ البرنامج
  • تظهر أخطاء منطقية أو قسمة على صفر
int a = 5, b = 0;
cout << a / b; // crash (Run-time)
4️⃣2️⃣ Exception

التعريف: طريقة لمعالجة الأخطاء بدون إيقاف البرنامج.

try {
    int a = 5, b = 0;
    if (b == 0) throw "Division by zero";
    cout << a / b;
}
catch (const char* msg) {
    cout << msg;
}


الأهمية:
  • أمان أعلى
  • تحكم أفضل في الأخطاء
4️⃣3️⃣ Casting

التعريف: تحويل نوع بيانات إلى نوع آخر.

int x = 5;
double y = (double)x;
أنواع مهمة:
  • Implicit (تلقائي)
  • Explicit (يدوي)
4️⃣4️⃣ Const Correctness

التعريف: استخدام const لمنع التعديل غير المقصود.

void print(const int x) {
    // x = 5; ❌
    cout << x;
}
الأهمية:
  • أمان
  • كود أوضح

ثاني عشر: توضيحات مهمة بأمثلة أقوى

🔴 Shallow Copy vs Deep Copy (بمثال حقيقي)
class Test {
public:
    int* x;
    Test(int v) {
        x = new int(v);
    }
};
Shallow Copy:
  • نسخ العنوان فقط
  • خطير

Deep Copy (الحل): 
Test(const Test& t) {
    x = new int(*t.x);
}
🔴 Stack vs Heap (مثال مقارن)
int a = 10;        // Stack
int* b = new int(20); // Heap
  • a يُحذف تلقائيًا
  • b يحتاج delete
🔴 Pointer vs Reference (مثال واضح)
int x = 10;
int* p = &x;
int& r = x;
  • Pointer يمكن تغييره
  • Reference لا يمكن أن يكون null

4️⃣5️⃣ Coupling

كلاس للحساب فقط
